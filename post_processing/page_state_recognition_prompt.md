# 页面与状态识别提示词

## 任务目标

分析 Android 手机界面截图，仅根据**当前这一张截图**识别**应用名**、**页面名**、**页面通用描述**与**本图差异化内容**，确保同一张截图多次处理得到完全一致的结果。不比较、不引用其他步骤或其它图片，只描述本图中可见的内容。

## 输出格式

你必须严格按照以下 JSON 格式输出，不要添加任何其他内容：

```json
{
  "app_name": "应用名称",
  "page_name": "页面名称",
  "page_description": "通用页面描述（不写具体内容，只描述页面有哪些类型的元素；不重复 page_name；5 句以内）",
  "differentiated_content": "本截图特有的具体内容（如具体商品、具体文案、具体状态值等；5 句以内）"
}
```

- **app_name**：当前界面所属应用的名称，中文或英文均可（如：微信、WeChat、Contacts、AudioRecorder、System）。
- **page_name**：当前页面的名称或类型（如：聊天页面、联系人列表、录音页面、桌面、系统设置）。可与应用组合用于区分，但此处仅填页面名/类型。
- **page_description**：**通用**描述，只描述这类页面通常有哪些**类型的**元素或区域，**不要写本截图的具体内容**。不必重复 page_name（如 page_name 已是「聊天页面」就不要再写「聊天界面，包含…」），直接写有哪些区域/元素即可；可写详细一些，5 句以内。
- **differentiated_content**：**本截图特有的具体内容**。例如：当前是哪个商品（八宝粥）、输入框里是什么字、当前联系人是谁、录音时长多少、列表里可见哪些具体项等。5 句以内。

## 页面识别规则

### 1. 应用与页面命名

- **app_name**：使用应用的中文或英文名称（根据截图中的实际显示）
  - 示例：微信、WeChat、Contacts、联系人、AudioRecorder、录音机、Settings、System
- **page_name**：描述当前界面的主要功能或内容类型
  - 示例：主页面、聊天页面、联系人列表、录音页面、设置页面、桌面

**组合示例**：
- app_name: `微信`，page_name: `聊天页面`
- app_name: `Contacts`，page_name: `联系人列表`
- app_name: `AudioRecorder`，page_name: `录音页面`
- app_name: `System`，page_name: `桌面`

### 2. 页面识别原则

1. **主要功能识别**：识别当前界面最主要的功能或内容区域
2. **应用识别**：通过界面元素、图标、标题栏等识别应用
3. **页面类型识别**：根据界面布局、主要元素判断页面类型（列表页、详情页、输入页、设置页、主页面等）

### 3. 特殊情况处理

- **系统桌面**：app_name: `System`，page_name: `桌面`
- **启动/加载页面**：page_name: `启动页面`
- **错误页面**：page_name: `错误页面`
- **权限请求/弹窗**：识别为当前页面，在 differentiated_content 中写明弹窗或权限请求的具体文案

## 字段填写规则（仅基于本张截图）

### 1. page_description（通用描述，必填）

**定义**：用 5 句以内的文字描述**这类页面通常包含哪些类型的元素或区域**，**不写本截图的具体内容**。

**原则**：
- **不重复 page_name**：page_name 已经标明页面类型（如「聊天页面」「商品详情」），此处**不要**再写「聊天界面，包含…」「商品详情页，包含…」等，直接写有哪些区域/元素即可。
- **泛化**：只写「有某某区域/元素」（如：有商品标题、商品图片、规格、价格；有消息列表、输入框、发送按钮；有列表、搜索框）。可写详细一些，多句描述布局与元素类型。
- **不写具体值**：不要出现具体商品名、具体联系人、具体文字内容、具体时间数值等，这些放在 differentiated_content。
- **自包含**：读者能据此想象这类页面的结构，而不是本图独有内容。

**示例**（注意不重复页面类型名，直接写结构）：
- 商品详情类：`顶部通常有商品主图或轮播。下方有商品标题、规格选择（如容量、口味）、价格与购买/加购按钮。可能还有详情 Tab、评价入口等。`
- 聊天类：`上方为消息列表区域，单条消息可为文本、图片等。底部有输入框、表情入口、发送或语音等按钮。键盘展开时可能遮挡部分消息。`
- 列表类：`顶部可有搜索框或筛选栏。主体为列表，每项可含头像、标题、副标题等。可能支持下拉刷新、上拉加载。`
- 录音类：`中央或底部有录音/暂停主按钮。常有时长显示、波形或进度。可能有已录文件列表、删除或重命名等操作。`
- 系统桌面：`顶部为状态栏（时间、电量、信号等）。主体为应用图标网格。底部可有常驻搜索栏或导航栏。`

### 2. differentiated_content（差异化内容，必填）

**定义**：用 5 句以内的文字描述**本张截图中可见的具体内容或状态值**。

**原则**：
- **具体**：写本图才有的信息（如：当前为八宝粥详情；输入框内容「你好」；与张三的对话；录音时长 00:15；列表可见 Wi-Fi、蓝牙等）。
- **仅依据本图**：不推测、不对比其它步骤。
- **可与 page_description 互补**：page_description 说「有什么类型的元素」，differentiated_content 说「这些元素在本图里具体是什么」。

**示例**：
- `当前商品为八宝粥，规格与价格已展示。`
- `输入框为空，键盘未展开。`
- `输入框内有文字「你好」，键盘已展开。`
- `正在录音，计时约 15 秒。`
- `列表已加载，可见多条联系人，无选中项。`
- `弹窗：权限请求，文案为「允许录音？」。`
- `系统桌面，显示时间与日期，可见 Gmail、Photos 等应用图标。`

### 3. app_name 与 page_name（必填）

- **app_name**：2～20 字，应用名称。
- **page_name**：2～20 字，页面类型/名称，与 app_name 配合唯一标识页面种类。

## 一致性保证

- **同一张截图多次识别**：应得到相同或语义一致的 app_name、page_name、page_description、differentiated_content。
- **仅依据本图**：不引用、不推测其他步骤或截图内容。
- **命名稳定**：应用名与页面类型用统一说法（如统一用「录音页面」而非混用「录音界面」）。
- **通用与具体分离**：page_description 只写通用结构，differentiated_content 只写本图具体内容。

## 边界情况

- **无法识别页面**：
  ```json
  {
    "app_name": "Unknown",
    "page_name": "未知页面",
    "page_description": "界面类型无法明确识别。",
    "differentiated_content": "界面内容无法明确识别。"
  }
  ```
- **页面明确、具体内容难归纳**：page_description 直接写通用元素与结构（不重复 page_name），differentiated_content 可写「当前可见状态」或「默认状态」等。

## 输出要求

1. **严格 JSON**：只输出 JSON，无解释、无 markdown 代码块标记。
2. **四个字段必填**：`app_name`、`page_name`、`page_description`、`differentiated_content` 均非空。
3. **page_description**：只写通用页面结构/元素类型，不重复 page_name，可写详细，5 句以内。
4. **differentiated_content**：只写本截图特有的具体内容或状态，5 句以内。

{"app_name": "微信", "page_name": "聊天页面", "page_description": "上方为消息列表区域，单条消息可为文本、图片等。底部有输入框、表情入口、发送或语音等按钮。键盘展开时可能遮挡部分消息。", "differentiated_content": "输入框为空，键盘未展开。"}
